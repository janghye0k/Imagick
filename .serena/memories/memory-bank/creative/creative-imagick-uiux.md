# 🎨 CREATIVE PHASE: UI/UX — Imagick

**상태**: 완료 (답변 반영 구체화)  
**기준**: `creative-imagick-questions.md` 답변 + package.json, tailwind 테마

---

## 1. 문제 정의 (Problem Statement)

- 브라우저 전용 이미지 편집 사이트의 **일관된 UI/UX**와 **전역 드래그 앤 드롭 업로드·파일 유지**를 설계한다.
- 랜딩, 네비, 도구별 화면, 다중 파일·용량 제한, 애니메이션·모션 라이브러리 사용 방식을 정한다.

---

## 2. 설계 결정 요약

### 2.1 사이트 구조·진입점

| 항목                         | 결정                                                                                                   |
| ---------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Vite 루트**                | 프로젝트 루트. `base/`는 참고용이며, 필요한 에셋·스타일은 **복사**하여 루트 구조에 반영.               |
| **첫 화면**                  | 랜딩: 소개 + 도구 카드 그리드.                                                                         |
| **도구 순서** (네비·카드 순) | 1. Convert → 2. Compress → 3. Resize → 4. Filter → 5. Crop → 6. Convert SVG → 7. Transform → 8. Sprite |
| **MVP 범위**                 | 위 8개 도구 전부 포함.                                                                                 |

### 2.2 네비게이션·레이아웃

| 항목          | 결정                                                      |
| ------------- | --------------------------------------------------------- |
| **네비**      | 상단 헤더 + 푸터 링크.                                    |
| **로고 클릭** | 항상 홈(/)으로 이동.                                      |
| **모바일**    | 햄버거 메뉴로 도구 접근.                                  |
| **다크 모드** | 제공. `base/src/styles/tailwind.css`의 `.dark` 테마 사용. |

### 2.3 전역 업로드·파일 유지 (핵심 UX)

| 항목                 | 결정                                                                                        |
| -------------------- | ------------------------------------------------------------------------------------------- |
| **파일 입력**        | 모든 페이지에서 **드래그 앤 드롭 + 클릭** 모두 지원.                                        |
| **편집 무관 페이지** | 랜딩 등 편집과 무관한 페이지에서 파일을 올리면 **Convert**로 자동 이동 후 해당 도구로 진행. |
| **페이지 전환**      | 도구를 바꿔도 **업로드된 파일 목록은 유지**. 전역 상태(또는 라우터+상태 조합)로 관리.       |
| **자연스러운 UX**    | 드롭 시 즉시 반응(로딩/성공 피드백), 리스트 표시, 제한 초과 시 안내 메시지.                 |

### 2.4 다중 파일·용량 제한

| 항목                 | 값    | 비고                                 |
| -------------------- | ----- | ------------------------------------ |
| **파일 개수 상한**   | 20개  | 초과 시 추가 업로드 불가, 안내 표시. |
| **파일당 최대 크기** | 50MB  | 초과 파일은 거절, 안내 표시.         |
| **전체 용량 상한**   | 100MB | 합계 초과 시 추가 업로드 불가.       |

(구현 시 상수로 두고 필요하면 조정.)

### 2.5 도구별 기능 범위

| 도구         | 결정                                                                           |
| ------------ | ------------------------------------------------------------------------------ |
| **Resize**   | 너비/높이(px), 비율(%), 비율 유지, 프리셋 등 **가능한 모든 옵션**.             |
| **Convert**  | **PNG, JPG, GIF, WebP, AVIF** 등 브라우저 지원 포맷 전부.                      |
| **Compress** | **슬라이더(0–100)** + **단계(낮음·중간·높음)** 둘 다.                          |
| **Filter**   | 밝기, 대비, 채도, 그레이스케일, 블러, 세피아, 색온도, 반전 등 **가능한 모두**. |

### 2.6 미리보기·다운로드

| 항목         | 결정                                                                                                         |
| ------------ | ------------------------------------------------------------------------------------------------------------ |
| **미리보기** | 데스크톱·모바일 각각 자연스럽게 (데스크톱: 옵션 오른쪽 또는 아래, 모바일: 세로 스택/탭 등). 반응형으로 배치. |
| **다운로드** | 미리보기 영역과 함께 배치. 다중 파일 시 일괄 다운로드(zip) 또는 개별 다운로드 제공.                          |

### 2.7 브랜딩·문구

| 항목                    | 결정                                                                                                    |
| ----------------------- | ------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------- |
| **사이트 타이틀 (SEO)** | `Imagick — Free Online Image Editor                                                                     | Resize, Convert, Compress in Browser` (package.json 기반). |
| **홈 한 줄 소개**       | `Edit images in your browser — no app, no sign-up. Resize, convert, compress, and more.` (직관적·친근). |

### 2.8 기술·성능

| 항목               | 결정                                                                                                                                                          |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **이미지 처리**    | 라이브러리 선호. **Canvas API가 더 적합한 부분은 Canvas** 사용. 그 외는 2025–2026 기준 유지보수·평가 좋은 라이브러리 (예: browser-image-compression 등) 선택. |
| **Web Worker**     | 큰 이미지 처리 시 **사용**. 메인 스레드 블로킹 방지.                                                                                                          |
| **파일 크기 제한** | 위 2.4와 동일 (파일당 50MB, 전체 100MB, 20개).                                                                                                                |

### 2.10 애니메이션·모션

| -------------- | ----------------------------------------------------------------------------------------------------------- |

## 3. 구현 가이드라인

### 3.1 프로젝트 구조 (Vite 루트 = 프로젝트 루트)

- `src/` 아래에 앱 소스. `base/`의 참고 파일은 **복사**하여 반영.
  - 예: `base/public/favicons` → `public/favicons`, `base/public/images` → `public/images`, `base/public/fonts` → `public/fonts`.
  - `base/src/styles/tailwind.css` → `src/styles/tailwind.css` 복사 후 경로만 조정.

### 3.2 전역 업로드 상태

- 전역 상태(React Context 또는 전역 store)에 **업로드된 파일 목록**과 **총 용량** 유지.
- 드래그 앤 드롭/파일 선택 시: 개수·용량 제한 검사 → 통과 시 목록에 추가, 실패 시 토스트/인라인 메시지.
- 라우팅: 편집 무관 페이지에서 업로드 발생 시 `navigate('/convert')` (또는 Convert 라우트) 후 해당 도구에서 동일 파일 목록 사용.

### 3.3 라우트 구조 (예시)

- `/` — 랜딩
- `/convert`, `/compress`, `/resize`, `/filter`, `/crop`, `/convert-svg`, `/transform`, `/sprite` — 도구별 페이지

### 3.4 반응형·미리보기

- 데스크톱: 옵션 패널 + 미리보기를 나란히 (또는 옵션 위·미리보기 아래).
- 모바일: 옵션 → 미리보기 순의 세로 스택, 또는 탭으로 전환. 다운로드 버튼은 미리보기 바로 아래 또는 고정 액션 바.

### 3.5 접근성

- 드롭 존·버튼에 적절한 라벨·aria 속성.
- 다크 모드: `prefers-color-scheme` 또는 토글로 `.dark` 적용.
